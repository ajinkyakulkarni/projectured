Description

ProjecturEd is a generic purpose projectional editor written in Common Lisp. Unfortunately due to lack of time it's under quite slow construction. Drop me a mail at projectured@gmail.com if you would like to participate.

Internals

If you are interested in the projectional editor internals, then currently the best thing to do is to look at the code. Surprisingly the code is not that long, it's less than 10,000 lines as of february, 2013. You can also read the files in the documentation folder of the repository, although it's far from being well structured. There are also a number of helpful pages in the wiki that might help getting some parts of the idea.

Screenshots

If you just want to take a look at the projectional editor in action, then there's a separate page in the wiki with a couple of screenshots showing various domains and projections.

Performance

Unfortunately performance is just not that great right now, although there are a few low hanging fruits in the code. The design also helps in that the printer and the reader are purely functional algorithms, so it wouldn't be that hard to utilize multiple cores for them. Finally both the printer and the reader are written using a constraint based change propagation library that allows lazy and incremental computation of results. One day it might even be fast enough to be useful...

Limitations

The editor is far from being usable at the moment, so don't expect too much right now. You have been warned!

Supported Backends

  • The SDL backend prints to a native window of the operating system, and it reads keyboard and mouse events.

  • The SLIME backend prints to a SLIME REPL in Emacs using SWANK, and it doesn't support reading events at all.

  • The WEB backend prints to the web browser using Ajax and JavaScript, and it reads keyboard and mouse events (not properly).

Supported Domains

There are a number of supported domains:

  • graphics
  • string
  • styled string
  • list
  • table
  • tree
  • XML
  • JSON
  • Java code
  • S-expression
  • Common Lisp code
  • book
  • widget
  • etc.

Supported Projections

There are a number of supported primitive projections:

  • string to styled string
  • styled string to graphics
  • line numbering
  • word wrapping
  • XML to tree
  • JSON to tree
  • Java to tree
  • Common Lisp to tree
  • book to tree
  • tree to string
  • list to string
  • table to string
  • widget to graphics
  • etc.

There are a number of supported generic projections:

  • remove
  • sort
  • focus
  • copy
  • etc.

There are a number of supported higher order projections:

  • sequential
  • parallel
  • recursive
  • nested
  • type dispatch
  • predicate dispatch
  • etc.

Testing

You should load the SDL backend via (LOAD-SYSTEM :PROJECTURED.SDL.TEST) and run the examples such as (TEST/EDITOR/JSON) from the :PRJECTURED.TEST package. You should immediately see a window popping up and showing some content, in this case it should be colorized text. You can move around with the cursor and edit some parts of the JSON document. Try clicking with the right mouse button to see what the editor knows about that location. Left clicking should put the cursor at the mouse location.

Some supported navigation gestures:

  • cursor keys
  • home and end keys (also control + home, control + end)
  • pageup and pagedown

Some editing gestures:

  • insert alphanumeric characters

Some supported projection changing gestures:

  • control + c (turns off coloring)
  • control + i (turns off indentation)
  • control + d (turns off delimiters)
  • control + s (turns off separators)
